# 12. 세그먼트트리와구간자료구조

## 변경 목적
- 구간 질의와 점/구간 업데이트를 빠르게 처리하는 자료구조를 정리한다.
- 처음 접하면 추상적으로 느껴질 수 있으므로, 목적과 필요 상황을 중심으로 이해한다.

## 이 폴더에서 다룰 내용
- 세그먼트 트리
- Lazy Propagation
- 펜윅 트리 기초
- 구간 질의와 업데이트 설계
- 평방 분할 기초

## 왜 중요한가
- 입력이 크고 질의가 많은 문제에서 브루트포스는 거의 항상 불가능하다.
- 상위권 코테와 심화 문제에서 자주 등장한다.

## 학습 목표
- 세그먼트 트리가 왜 필요한지 설명할 수 있다.
- 구간 합/최솟값 질의를 세그먼트 트리로 처리할 수 있다.
- 업데이트 종류에 따라 어떤 자료구조가 필요한지 구분할 수 있다.

## 세부 분류
### 1. 세그먼트 트리 기본
- 트리 구조
- 구간 분할
- 질의 처리
- 점 업데이트

### 2. Lazy Propagation
- 구간 업데이트
- 나중에 전파하는 이유
- push 타이밍 이해

### 3. 다른 구간 자료구조
- 펜윅 트리
- sqrt decomposition

## 자주 하는 실수
- 노드가 담당하는 구간 의미를 놓치는 실수
- 인덱스 기준이 꼬이는 실수
- Lazy 값을 언제 전파해야 하는지 헷갈리는 실수
- 세그먼트 트리가 필요 없는 문제에 과하게 쓰는 실수

## 대표 문제 유형
- 구간 합 질의
- 구간 최솟값/최댓값
- 점 업데이트 + 구간 질의
- 구간 업데이트 + 구간 질의

## 정리 포인트
- 각 노드가 무엇을 저장하는지 먼저 적는다.
- 질의 함수와 업데이트 함수의 반환 의미를 명확히 적는다.
- 배열 인덱스 기준과 트리 인덱스 기준을 분리해서 본다.

## 체크리스트
- [ ] 세그먼트 트리의 목적을 설명할 수 있다.
- [ ] 점 업데이트/구간 질의를 구현할 수 있다.
- [ ] Lazy Propagation이 필요한 상황을 이해한다.
- [ ] 펜윅 트리와 세그먼트 트리 차이를 대략 설명할 수 있다.

## 추천 학습 순서
1. 세그먼트 트리 기본
2. 점 업데이트 + 구간 질의
3. Lazy Propagation
4. 펜윅 트리
5. 평방 분할

## 복습 기준
- 질의 수가 많고 업데이트가 섞이면 구간 자료구조를 떠올릴 수 있어야 한다.
