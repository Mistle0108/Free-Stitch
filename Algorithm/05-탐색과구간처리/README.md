# 05. 탐색과구간처리

## 변경 목적
- 배열과 수열 문제를 효율적으로 처리하는 핵심 기법을 정리한다.
- 선형 탐색에서 끝내지 않고, 구간 정보와 조건 판별을 이용하는 감각을 키운다.

## 이 폴더에서 다룰 내용
- 이분 탐색
- 누적 합
- 투 포인터
- 슬라이딩 윈도우
- 스위핑 기초

## 왜 중요한가
- 코딩 테스트에서 "구간", "범위", "조건을 만족하는 최솟값/최댓값" 문제는 매우 자주 나온다.
- 이 파트는 정렬, 해시, 그래프 못지않게 실전성이 높다.

## 학습 목표
- 답을 직접 찾는 대신 조건을 판별하는 문제로 바꿔 생각할 수 있다.
- 누적 합으로 구간 합을 빠르게 계산할 수 있다.
- 투 포인터로 부분 배열/부분 수열 문제를 다룰 수 있다.

## 세부 분류
### 1. 이분 탐색
- 정렬된 배열에서 값 찾기
- lower bound / upper bound 감각
- 파라메트릭 서치

### 2. 누적 합
- 구간 합 계산
- 2차원 누적 합
- 전처리 후 빠른 질의 처리

### 3. 투 포인터 / 슬라이딩 윈도우
- 연속 구간 탐색
- 합/길이/조건 만족 구간 찾기
- 윈도우 유지와 갱신

### 4. 스위핑
- 이벤트 순서대로 처리
- 구간 겹침
- 좌표축 기반 정렬 처리

## 자주 하는 실수
- 이분 탐색의 종료 조건과 mid 갱신을 틀리는 실수
- 누적 합 인덱스를 하나 밀리게 계산하는 실수
- 투 포인터에서 포인터 이동 조건을 애매하게 잡는 실수
- 스위핑에서 정렬 기준을 잘못 두는 실수

## 대표 문제 유형
- 특정 값 존재 여부 찾기
- 최대/최소 가능값 찾기
- 구간 합 질의
- 연속 부분 수열 합
- 조건을 만족하는 최소 길이 구간
- 선분/일정 겹침 계산

## 정리 포인트
- "정답을 찾는 문제"인지, "가능 여부를 판정하는 문제"인지 먼저 구분한다.
- 누적 합은 배열 정의와 인덱스 기준을 같이 적는다.
- 투 포인터는 어떤 조건에서 left/right가 움직이는지 문장으로 먼저 적는다.

## 체크리스트
- [ ] 이분 탐색을 반복문으로 안정적으로 구현할 수 있다.
- [ ] lower bound / upper bound 개념을 이해한다.
- [ ] 1차원 누적 합을 바로 만들 수 있다.
- [ ] 연속 구간 문제에서 투 포인터를 떠올릴 수 있다.
- [ ] 문제를 보고 파라메트릭 서치 가능성을 판단할 수 있다.

## 복습 기준
- 구간 관련 문제가 나오면 브루트포스보다 먼저 누적 합/투 포인터/이분 탐색을 떠올릴 수 있어야 한다.
