# 04. 정렬과기본자료구조

## 변경 목적
- 문제 해결에 자주 쓰이는 기본 자료구조와 정렬 활용 패턴을 정리한다.
- 단순히 자료구조 이름을 아는 수준이 아니라, 언제 선택해야 하는지 기준을 만든다.

## 이 폴더에서 다룰 내용
- 정렬
- 집합(Set)
- 맵(Map) / 해시
- 스택
- 큐
- 덱
- 우선순위 큐 / 힙

## 왜 중요한가
- 많은 문제는 "어떤 자료구조를 쓰는가"가 핵심이다.
- 정렬과 해시는 코테에서 등장 빈도가 매우 높다.

## 학습 목표
- 배열을 그냥 순회할지, 정렬할지, 해시를 쓸지 판단할 수 있다.
- 스택/큐/덱의 차이를 실제 문제 상황에 연결할 수 있다.
- 우선순위 큐가 필요한 문제를 알아볼 수 있다.

## 세부 분류
### 1. 정렬
- 오름차순/내림차순
- 커스텀 비교
- 정렬 후 인접 비교
- 정렬 기반 그리디

### 2. 해시와 맵
- 빈도 수 세기
- 존재 여부 확인
- 키-값 매핑
- 중복 제거

### 3. 스택/큐/덱
- LIFO / FIFO / 양방향 처리
- 괄호 검사
- 순서 유지
- 회전/슬라이딩 처리

### 4. 우선순위 큐
- 최솟값/최댓값 빠르게 꺼내기
- 작업 우선순위 관리
- 다익스트라 같은 알고리즘의 기반

## 자주 하는 실수
- 정렬이 필요한데도 순회만으로 풀려는 실수
- 해시를 쓸 수 있는 문제를 이중 반복문으로 처리하는 실수
- 스택과 큐를 헷갈려 자료 흐름이 꼬이는 실수
- 우선순위 큐의 정렬 기준을 잘못 설정하는 실수

## 대표 문제 유형
- 숫자 정렬
- 좌표 정렬
- 빈도 수 계산
- 중복 제거
- 괄호 검사
- 카드/문서 대기열
- 최소 힙/최대 힙 문제

## 추천 정리 방식
```md
## 자료구조/기법 이름

### 언제 쓰는가
- 문제의 신호

### 장점
- 왜 이 선택이 유리한가

### 주의점
- 구현 시 실수 포인트

### 대표 문제
- 다시 풀 문제 목록
```

## 체크리스트
- [ ] 정렬이 필요한 문제의 특징을 말할 수 있다.
- [ ] 해시로 시간복잡도를 줄이는 대표 패턴을 안다.
- [ ] 스택/큐/덱을 문제 흐름에 맞게 구분해서 선택할 수 있다.
- [ ] 우선순위 큐의 필요성을 예시로 설명할 수 있다.
