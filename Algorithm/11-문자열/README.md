# 11. 문자열

## 변경 목적
- 문자열 처리 문제를 단순 구현부터 패턴 탐색까지 단계적으로 정리한다.
- 문자열을 배열처럼 다루는 수준을 넘어, 검색과 패턴 문제 접근법을 익힌다.

## 이 폴더에서 다룰 내용
- 문자열 기초 처리
- 문자열 비교와 정렬
- 패턴 매칭 기초
- KMP
- Trie 기초
- 문자열 해싱 입문

## 왜 중요한가
- 문자열 문제는 구현처럼 보이지만, 알고리즘 선택이 필요한 경우가 많다.
- 상위권 대비로 갈수록 별도 문자열 알고리즘의 중요도가 커진다.

## 학습 목표
- 문자열 순회/분리/비교를 안정적으로 다룰 수 있다.
- 패턴 찾기 문제에서 단순 탐색과 효율적 탐색을 구분할 수 있다.
- KMP와 Trie의 목적을 설명할 수 있다.

## 세부 분류
### 1. 문자열 기초
- 문자 분해
- 부분 문자열
- 치환
- 비교

### 2. 문자열 자료구조/알고리즘
- 접두사/접미사 개념
- KMP의 pi 배열
- Trie 노드 구조
- 해싱 기초

## 자주 하는 실수
- 인덱스 처리에서 부분 문자열 범위를 틀리는 실수
- 문자열 비교 기준을 잘못 이해하는 실수
- KMP를 원리 없이 외워서 응용이 안 되는 실수
- Trie 구현에서 메모리 구조를 막연하게 잡는 실수

## 대표 문제 유형
- 문자열 개수 세기
- 특정 패턴 등장 여부
- 접두사/접미사 판별
- 문자열 집합
- 단어 검색

## 추천 정리 방식
```md
## 알고리즘/개념 이름

### 목적
- 무엇을 빠르게 하려는가

### 핵심 개념
- 접두사, 실패 함수, 노드 등

### 구현 포인트
- 자료구조와 순회 방식

### 활용 문제
- 어디에 쓰이는가
```

## 체크리스트
- [ ] 기본 문자열 처리 문제를 안정적으로 풀 수 있다.
- [ ] 접두사/접미사 개념을 설명할 수 있다.
- [ ] KMP의 목적과 흐름을 이해한다.
- [ ] Trie가 필요한 상황을 예시로 말할 수 있다.

## 추천 학습 순서
1. 문자열 기초 구현
2. 문자열 정렬/비교
3. 패턴 매칭 기초
4. KMP
5. Trie
6. 문자열 해싱 입문

## 복습 기준
- 문자열 문제를 봤을 때 "그냥 구현", "자료구조 필요", "패턴 알고리즘 필요"를 구분할 수 있으면 좋다.
