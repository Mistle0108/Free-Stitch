# 01. 기초구현

## 변경 목적
- 알고리즘 공부의 출발점이 되는 문법 적응력과 구현 안정성을 다진다.
- 단순해 보이지만 실수하기 쉬운 입력 처리, 조건 분기, 반복문, 배열/문자열 조작을 확실히 익힌다.

## 이 폴더에서 다룰 내용
- 입력과 출력
- 조건문과 반복문
- 1차원 배열
- 2차원 배열
- 문자열 기초
- 구현 문제에서의 예외 처리

## 왜 중요한가
- 코딩 테스트에서 막히는 원인이 알고리즘 자체보다 구현 실수인 경우가 많다.
- 기초 구현이 약하면 DFS, DP 같은 상위 주제도 코드가 불안정해진다.
- 이 파트는 이후 모든 폴더의 기본 체력 역할을 한다.

## 학습 목표
- 문제를 읽고 입력 형식을 빠르게 코드로 옮길 수 있다.
- 반복문과 조건문을 사용할 때 경계값 실수를 줄일 수 있다.
- 문자열과 배열을 자유롭게 순회하고 가공할 수 있다.
- 2차원 좌표 처리에서 인덱스와 범위 검사를 안정적으로 할 수 있다.

## 세부 분류
### 1. 입출력
- 빠른 입력이 필요한 상황 구분
- 여러 줄 입력 처리
- 공백 기준 파싱

### 2. 조건문/반복문
- 분기 누락 방지
- 중첩 반복문의 역할 분리
- break, continue 사용 판단

### 3. 배열과 문자열
- 인덱스 접근
- 부분 구간 처리
- 문자/숫자 변환
- 빈도 계산

### 4. 2차원 구현
- 좌표 이동
- 범위 체크
- 행/열 순회
- 보드 시뮬레이션

## 자주 하는 실수
- 인덱스 범위를 하나 초과하거나 덜 도는 실수
- 문자열 길이나 배열 길이를 반복해서 계산하다가 조건을 틀리는 실수
- 입력 형식을 정확히 안 읽고 예외 케이스를 놓치는 실수
- 2차원 배열에서 x/y와 row/col을 혼동하는 실수

## 대표 문제 유형
- 그대로 출력하기
- 숫자/문자열 누적 처리
- 최댓값/최솟값 찾기
- 문자 개수 세기
- 표나 보드에서 조건에 맞는 칸 찾기
- 구현 위주의 시뮬레이션

## 추천 정리 방식
- 문제를 풀고 끝내지 말고, 아래 항목을 함께 기록한다.

### 기록 템플릿
```md
## 문제 이름

### 핵심
- 이 문제에서 중요한 구현 포인트

### 배운 점
- 새로 익힌 문법 또는 패턴

### 실수
- 내가 틀린 이유

### 다시 풀 때 체크할 것
- 다음에는 무엇을 먼저 확인할지
```

## 체크리스트
- [ ] 입력 형식을 보고 바로 자료형과 저장 구조를 정할 수 있다.
- [ ] 반복문 경계값을 스스로 검증할 수 있다.
- [ ] 문자열 순회, 치환, 변환을 자연스럽게 할 수 있다.
- [ ] 2차원 배열 범위 검사를 습관적으로 넣을 수 있다.
- [ ] 구현 문제를 디버깅할 때 출력 로그 포인트를 스스로 잡을 수 있다.
