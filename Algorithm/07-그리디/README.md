# 07. 그리디

## 변경 목적
- 매 순간의 최선 선택이 전체 최적해로 이어지는 문제를 정리한다.
- 감으로 푸는 것이 아니라, 선택 기준과 정당화를 함께 기록하는 습관을 만든다.

## 이 폴더에서 다룰 내용
- 정렬 기반 그리디
- 선택 기준 설계
- 반례 찾기
- 교환 논증 감각

## 왜 중요한가
- 구현은 간단하지만, 왜 맞는지 설명하기 어려운 경우가 많다.
- 실전 코테에서 빈도는 높고, 직관만으로 접근하면 틀리기 쉬운 주제다.

## 학습 목표
- 문제에서 그리디 신호를 읽을 수 있다.
- 선택 기준을 한 줄 규칙으로 정리할 수 있다.
- 반례를 통해 잘못된 기준을 걸러낼 수 있다.

## 세부 분류
### 1. 정렬 기반 선택
- 시작/종료 시각 정렬
- 비용 기준 정렬
- 큰 값/작은 값 우선 선택

### 2. 누적 선택
- 현재 가능한 최선 갱신
- 조건 만족 범위 유지

### 3. 반례 검증
- 단순 직관이 틀릴 수 있는 경우 찾기
- 선택 기준이 깨지는 입력 만들기

## 자주 하는 실수
- 정당화 없이 "가장 커 보이는 값"만 고르는 실수
- 정렬 기준을 잘못 잡는 실수
- 현재 최선과 전체 최선을 혼동하는 실수
- 반례 검증 없이 제출하는 실수

## 대표 문제 유형
- 회의실 배정
- 동전 문제
- 로프
- 최소 비용/최대 개수 선택
- 구간 선택

## 정리 포인트
- 왜 이 선택이 항상 유리한지 문장으로 남긴다.
- 최소한 한 개 이상의 반례 후보를 직접 써 본다.
- 정렬 기준과 tie-breaker를 분리해서 적는다.

## 체크리스트
- [ ] 그리디 문제의 대표 신호를 말할 수 있다.
- [ ] 선택 기준을 수식 또는 문장으로 설명할 수 있다.
- [ ] 정렬 기준을 바꾸면 왜 틀리는지 반례를 만들 수 있다.
- [ ] 구현 전에 정당화를 먼저 적는 습관이 있다.

## 복습 기준
- 비슷한 문제를 만났을 때 "정렬 기준 + 선택 규칙 + 정당화"를 세트로 떠올릴 수 있으면 좋다.
