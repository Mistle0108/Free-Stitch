# 06. 백트래킹

## 변경 목적
- 완전탐색을 더 효율적으로 수행하는 백트래킹의 구조를 정리한다.
- 경우의 수를 모두 보되, 불필요한 탐색을 잘라내는 기준을 익힌다.

## 이 폴더에서 다룰 내용
- 순열과 조합 생성
- 중복 조합/중복 순열
- 조건 기반 탐색
- 가지치기

## 왜 중요한가
- 백트래킹은 단순 구현이 아니라 "상태를 만들고 되돌리는 흐름"을 이해해야 한다.
- DFS와 함께 많이 나오며, 문제 해결 사고력을 키워 준다.

## 학습 목표
- 재귀 기반 탐색 구조를 직접 설계할 수 있다.
- 방문 배열, 선택 배열, 현재 경로를 적절히 관리할 수 있다.
- 가지치기가 왜 필요한지 설명할 수 있다.

## 세부 분류
### 1. 순열/조합 생성
- 방문 배열 활용
- 시작 인덱스 조절
- 중복 방지

### 2. 조건 만족 탐색
- 합 조건
- 배치 조건
- 순서 조건

### 3. 가지치기
- 현재 상태만 보고 불가능한 경우 중단
- 최적값보다 나빠질 것이 확실한 경우 중단

## 자주 하는 실수
- 재귀 호출 후 상태 복원을 빠뜨리는 실수
- 중복된 경우를 여러 번 세는 실수
- 종료 조건과 탐색 조건을 섞어서 코드가 복잡해지는 실수
- 가지치기를 넣으면서 정답을 놓치는 실수

## 대표 문제 유형
- N과 M 시리즈
- 스도쿠
- 연산자 끼워넣기
- 부분 수열/부분 집합 탐색
- 배치 문제

## 추천 정리 방식
```md
## 문제 이름

### 상태
- 재귀 함수 인자

### 종료 조건
- 언제 답을 기록하는지

### 선택지
- 다음에 무엇을 시도하는지

### 가지치기
- 어떤 경우를 중단하는지

### 복원
- 호출 후 되돌리는 값
```

## 체크리스트
- [ ] 순열과 조합 생성 코드를 직접 작성할 수 있다.
- [ ] 백트래킹 함수의 인자를 왜 그렇게 두는지 설명할 수 있다.
- [ ] 상태 복원의 중요성을 알고 있다.
- [ ] 가지치기를 넣을 때 정답 누락 위험을 점검할 수 있다.
