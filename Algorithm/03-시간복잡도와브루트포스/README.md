# 03. 시간복잡도와브루트포스

## 변경 목적
- 문제를 풀기 전에 "이 풀이가 시간 안에 가능한가"를 판단하는 습관을 만든다.
- 완전탐색, 재귀, 분할정복의 기본 사고방식을 정리한다.

## 이 폴더에서 다룰 내용
- 시간복잡도
- 공간복잡도
- 브루트포스
- 재귀
- 분할정복

## 왜 중요한가
- 알고리즘 선택은 구현보다 먼저 이루어져야 한다.
- 문제를 보자마자 가능한 탐색 범위를 가늠할 수 있어야 효율적인 풀이를 고를 수 있다.

## 학습 목표
- 입력 크기를 보고 허용 가능한 복잡도를 추정할 수 있다.
- 완전탐색이 가능한 문제와 불가능한 문제를 구분할 수 있다.
- 재귀 호출 구조를 트리처럼 이해할 수 있다.
- 분할정복이 반복문보다 자연스러운 상황을 구분할 수 있다.

## 세부 분류
### 1. 시간복잡도
- `O(1)`, `O(log N)`, `O(N)`, `O(N log N)`, `O(N^2)` 감각
- 입력 크기 대비 허용 연산 수 추정

### 2. 브루트포스
- 모든 경우 확인
- 중복 없는 탐색
- 조건 만족 여부 검사

### 3. 재귀
- 종료 조건
- 문제를 더 작은 문제로 나누는 방식
- 호출 흐름 추적

### 4. 분할정복
- 절반으로 나누기
- 결과를 합치기
- 대표 예시: 병합정렬, 거듭제곱 분할

## 자주 하는 실수
- 입력 크기를 무시하고 단순 구현부터 시작하는 실수
- 재귀 종료 조건을 빠뜨리는 실수
- 브루트포스에서 중복 경우를 여러 번 세는 실수
- 분할 이후 병합 단계의 조건을 놓치는 실수

## 대표 문제 유형
- 가능한 모든 조합/순열 확인
- 재귀 호출로 별/패턴 출력
- 분할정복 기반 계산
- 모든 경우를 돌며 최적값 찾기

## 정리 포인트
- 문제마다 입력 크기와 복잡도 기준을 먼저 적는다.
- "이 문제는 왜 브루트포스로 가능한가/불가능한가"를 한 줄로 적는다.
- 재귀는 호출 흐름 그림을 함께 남긴다.

## 기록 템플릿

## 체크리스트
- [ ] 입력 크기만 보고 대략적인 허용 복잡도를 말할 수 있다.
- [ ] 브루트포스 가능 여부를 빠르게 판단할 수 있다.
- [ ] 재귀 함수 작성 시 종료 조건을 먼저 적는 습관이 있다.
- [ ] 분할정복의 구조를 코드 없이 설명할 수 있다.
