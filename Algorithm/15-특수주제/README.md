# 15. 특수주제

## 변경 목적
- 일반 취업 준비 범위를 넘어서는 특수 알고리즘이나 대회형 주제를 모아 관리한다.
- 지금 당장 필수는 아니더라도, 나중에 확장 학습할 수 있는 저장소 역할을 하게 한다.

## 이 폴더에서 다룰 내용
- 인터랙티브 문제
- 게임 이론
- FFT
- 랜덤화/확률 아이디어
- 기타 특수한 알고리즘 주제

## 왜 중요한가
- 빈도는 낮지만, 특정 문제군에서는 별도 접근법이 필수다.
- 공부 범위를 관리하기 위해 "지금 안 해도 되는 것"을 분리해 두는 의미가 있다.

## 학습 목표
- 특수 주제를 기본 주제와 구분해서 관리할 수 있다.
- 나중에 확장 학습이 필요할 때 참고 기준을 갖는다.

## 세부 분류
### 1. 인터랙티브
- 질의/응답 방식
- 출력 플러시 개념
- 로컬 테스트 전략

### 2. 게임 이론
- 승패 상태
- Sprague-Grundy 기초

### 3. FFT 및 고급 수학 도구
- 빠른 다항식 곱셈의 목적
- 적용 가능한 문제 형태

## 자주 하는 실수
- 아직 필요한 수준이 아닌데 무리하게 먼저 공부해 흐름이 끊기는 실수
- 개념 목적 없이 구현만 외우는 실수
- 특수 문제를 일반 문제처럼 접근하는 실수

## 추천 운영 방식
- 이 폴더는 초반에 깊게 파지 않아도 된다.
- 기본 주제를 충분히 익힌 뒤, 필요한 주제만 추가로 정리한다.
- 각 주제마다 "언제 필요한가"를 중심으로 기록한다.

## 기록 템플릿
```md
## 주제 이름

### 무엇을 해결하는가
- 문제 목적

### 언제 필요한가
- 등장 신호

### 선행 지식
- 먼저 알아야 하는 것

### 지금 수준에서의 우선순위
- 높음 / 중간 / 낮음
```

## 체크리스트
- [ ] 이 폴더가 기본 학습과 심화 학습을 분리하는 용도라는 점을 이해한다.
- [ ] 지금 당장 필요한 특수 주제와 아닌 주제를 구분할 수 있다.
- [ ] 새 주제를 정리할 때 목적과 선행 지식을 함께 기록한다.

## 복습 기준
- 특수주제는 "완벽 숙달"보다 "존재를 알고, 필요할 때 다시 찾아올 수 있는 상태"면 충분하다.
