# 13. 고급그래프

## 변경 목적
- 기본 그래프를 넘어서는 고급 그래프 알고리즘을 분리 정리한다.
- 취업 준비 기본 범위를 넘어서더라도, 상위 난도 대비용으로 축적할 수 있게 만든다.

## 이 폴더에서 다룰 내용
- 강한 연결 요소(SCC)
- 이분 매칭
- 네트워크 플로우
- 최소 비용 최대 유량
- 고급 그래프 모델링

## 왜 중요한가
- 일반 코테에서는 비중이 낮을 수 있지만, 어려운 문제와 상위권 대비에서 차이를 만든다.
- 기본 그래프 이해가 단단할수록 이 파트 흡수가 쉬워진다.

## 학습 목표
- SCC, 매칭, 플로우가 각각 어떤 문제를 해결하는지 구분할 수 있다.
- 그래프 문제를 "흐름", "매칭", "연결 성분" 관점으로 재해석할 수 있다.

## 세부 분류
### 1. SCC
- 방향 그래프의 강한 연결
- Kosaraju / Tarjan 개요

### 2. 이분 매칭
- 두 그룹 간 연결
- 최대 매칭

### 3. 네트워크 플로우
- 소스/싱크
- 용량(capacity)
- 증가 경로

### 4. 최소 비용 최대 유량
- 비용과 유량을 동시에 고려

## 자주 하는 실수
- 문제를 일반 BFS/DFS 수준에서만 보다가 플로우 모델링 기회를 놓치는 실수
- 그래프 방향성과 용량 의미를 섞는 실수
- 알고리즘 이름만 외우고 모델링 기준을 모르는 실수

## 대표 문제 유형
- 방향 그래프 압축
- 작업 배정
- 최대 매칭
- 유량 계산
- 경로 선택 비용 최적화

## 정리 포인트
- 알고리즘 구현보다 "언제 이 모델로 바꾸는가"를 먼저 적는다.
- 노드/간선/용량/비용의 의미를 문제 문장 기준으로 기록한다.

## 체크리스트
- [ ] SCC의 목적을 설명할 수 있다.
- [ ] 이분 매칭 문제의 형태를 알아볼 수 있다.
- [ ] 네트워크 플로우 기본 용어를 이해한다.
- [ ] 플로우 문제를 모델링하는 감각을 조금씩 키우고 있다.

## 추천 학습 순서
1. SCC
2. 이분 매칭
3. 네트워크 플로우
4. 최소 비용 최대 유량

## 복습 기준
- 이 파트는 구현 숙련보다 모델링 이해가 더 중요하다. 처음에는 알고리즘 목적과 문제 신호를 정리하는 데 집중하면 된다.
